Hello, today I will be comparing two of the most used backend languages - PHP and JavaScript(Node.js). I will talk about their advantages and disadvantages, as well as best use cases. But first I want to clarify what a backend language is.

1. What is a backend language

Backend, also server-side development is a portion of programming focused on storing and managing data like user accounts, cookies, blog posts, etc. That data is then provided to a website through number of data transfer protocols, the most common one is the HTTP protocol. So a backend language has to be able to at least receive and respond to user requests, communicate with a database and access the servers filesystem.

2. What will be rated

  • - performance
  • - simplicity
  • - syntax
  • - features
  • - hosting options

3. The comparison

PHP

PHP is a web scripting language created in 1995 and is the most used backend language today. It is so popular because of it's ease of use and wide variety of features. However PHP is just a scripting language and it is not capable of running as a web server by itself - you need Apache or a different http server in order to use it. This also removes some features like WebSocket communication or memory storage.

Example PHP code:

      
  <?php
    $message = "Hello world!";
    echo $message;
  ?>
Pros:
  • - it is very easy to use
  • - it has many additional features out of the box
  • - it is easy to host for free
Cons:
  • - it only runs when a request is made
  • - the syntax is unusual with lots of special characters
  • - it has inconsistent naming of functions and utilities

Node.js

Node.js is a "JavaScript runtime environment" created in 2009. It uses JavaScript as it's programming language. Unlike PHP it works as an http server, so there is no need for additional software.

Example Node.js code:

      
    const http = require('http');
    const message = "Hello world!";

    http.createServer((req, res) => {
      res.write(message);
      res.end();
    }).listen(8080);
Pros:
  • - you don't need additional software for the http server
  • - it has a big community of developers supporting it and creating new packages
  • - it is written in JavaScript - you can use one language both on frontend and backend
Cons:
  • - it is much more difficult to host online
  • - it is harder to start with

4. Conclusion

I personally prefer Node.js, because it is more modern, I like JavaScript and I like to have low level control over the http server. However it all depends on the project, you're building - if you're building a simple web app, or website, I would probably recommend PHP, maybe even WordPress.
